## 常見失敗模式

1. **HTML selector 過期**: `data.gov.tw` 改版,`requests.get(...).text` 拿到的 HTML 結構變了,
   `BeautifulSoup` 找不到對應 tag。
   - 修法:用 `data-*` 屬性或更穩定的 regex。請先 `curl` 一次當前網頁確認結構。

2. **CSV 編碼變動**:從 Big5 變 UTF-8、或加上 BOM。
   - 修法:在 `download.py` 加上編碼偵測(`chardet` 或 `charset_normalizer`)。

3. **TLS / 憑證問題**:政府網站偶爾換 cert。
   - 修法:`verify=False` 是暫時手段,更好的做法是更新 `certifi`。

4. **URL 改路徑**:dataset 從 `/dataset/35321` 換成 `/dataset/xxxxx`。
   - 修法:把所有 hardcoded URL 集中到 `download.py` 的常數。

## 你的工作流程

1. 讀 ISSUE body(失敗訊息 + traceback + log excerpt)
2. 讀 `scripts/download.py` 與 `scripts/parse_road_csv.py` 找出對應錯誤點
3. `curl` 一次當前 `data.gov.tw/dataset/35321` 的 HTML,確認結構
4. 修改 selector / 解析邏輯
5. 新增/調整 unit test 到 `scripts/tests/`(如果沒有這個目錄就建)
6. 跑 `python scripts/main.py --dry-run`(如果支援)或 `python scripts/download.py` 驗證
7. 確認 `data/latest.csv` 內容合理(有 header、有資料行)
8. `git checkout -b auto-heal/issue-<NUM>`
9. `git add scripts/ data/`(注意不要 `git add .`)
10. `git commit -m "fix: <一句話描述> (closes #<NUM>)"`
11. `git push -u origin auto-heal/issue-<NUM>`
12. 用 `gh pr create` 開 PR,body 引用 ISSUE
